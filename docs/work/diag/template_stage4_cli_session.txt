# Stage4 CLI session template

Target env:
- `python3 -m platformio run -e luce_stage4 -t upload --upload-port /dev/cu.usbserial-0001`
- `python3 -m platformio device monitor -p /dev/cu.usbserial-40110`

Suggested interaction:

1. Upload and monitor with:
   - `python3 -m platformio run -e luce_stage4 -t upload --upload-port /dev/cu.usbserial-0001`
   - `python3 -m platformio device monitor -p /dev/cu.usbserial-40110`
2. On first boot, capture:
   - `LUCE STAGE4`
   - build/feature lines
   - boot diagnostics and `CLI listening on UART0...`
3. Run commands:
   - `help`
   - `status`
   - `nvs_dump`
   - `i2c_scan`
   - `mcp_read gpioa`
   - `mcp_read gpiob`
   - `buttons`
   - `relay_mask 0x55`
   - `relay_set 0 1`
   - `relay_set 0 0`
   - `lcd_print LUCE Stage4 CLI`
   - `reboot`

What to paste back:
- Paste exact monitor log with command results after each entry, including:
  - command arguments
  - `rc=<result>`
  - `status:` lines for runtime checks

Sample lines to expect (non-exhaustive):
- `LUCE STAGE4 ...`
- `CLI listening on UART0 at 115200. Type 'help' for commands.`
- `CLI cmd='status' argc=1`
- `status: stage=4 ...`
- `CLI command relay_set: ch=0 value=1 ... rc=ESP_OK`
- `CLI command relay_mask: mask=0x55 rc=ESP_OK`
- `CLI command mcp_read gpioa rc=ESP_OK value=...`
- `CLI command reboot: restarting`
